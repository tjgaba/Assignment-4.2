# Story #[1]: [Basic room booking]
**As a** [Employee]
**I want to** [book a conference room]
**So that** [i can conduct work shops with my team]

# Story #[2]: [Recurring meetings setup]
**As a** [Employee]
**I want to** [select a automation feature ]
**So that** [recurring meetings could automatically be booked]

# Story #[3]: [Room capacity filtering]
**As a** [Employee]
**I want to** [select the number of personnel who would be attending meetings]
**So that** [i can know before-hand how much catering is to be procured]

# Story #[4]: [Booking cancellation]
**As a** [Employee]
**I want to** [be able to cancel the booking]
**So that** [i can avoid last minute change of plan due to funds or other commitments]

# Story #[5]: [Room equipment requirements]
**As a** [Employee]
**I want to** [see or view list of equipment provided with the bookings]
**So that** [so that i can select which equipment will be necessary to procure to avoid unnecessary expenditure]

# Story #[6]: [Admin dashboard viewing]
**As a** [Admin]
**I want to** [be able to view the dashboard]
**So that** [i can see how many bookinigs where done for a certain day or period]

# Story #[7]: [Room maintenance scheduling]
**As a** [Facilities-Manager]
**I want to** [be notified on my phone]
**So that** [i know the time when a conference room is currently being used or done with usage]

# Story #[8]: [Visitor booking assistance]
**As a** [Receptionist]
**I want to** [an imbeded tutoring feature on the UI system that will guide the employee step-by-step]
**So that** [the workload of the receptionist could be lighter.]

# Story #[9]: [Booking conflict resolution]
**As a** [admin]
**I want to** [an implimented feature that will automatically send a link to the employee’s phone via push notification]
**So that** [they can confirm the booking to avoid duplicates]

# Story #[10]: [Usage reports generation]
**As a** [admin]
**I want to** [want drop-down feature that will generate usage reportss]
**So that** [so that estimates could be recieved according to categories of equipments, User names, room number and time frames]

# Conference Room Booking System – Acceptance Criteria
---
## Story #1: Basic room booking
### Acceptance Criteria:
- [ ] Given a booking is completed, When the process finishes, Then a confirmation message is displayed to the employee.
---
## Story #2: Recurring meetings setup
### Acceptance Criteria:
- [ ] Given recurrence settings are defined, When the booking is saved, Then all recurring meetings are automatically created.
---
## Story #3: Room capacity filtering
### Acceptance Criteria:
- [ ] Given an employee is booking a room, When they enter the number of attendees, Then only rooms with sufficient capacity are shown.
---
## Story #4: Booking cancellation
### Acceptance Criteria:
- [ ] Given an employee has an active booking, When they choose to cancel it, Then the booking is removed from the schedule.
---
## Story #5: Room equipment requirements
### Acceptance Criteria:
- [ ] Given an employee views room details, When equipment information is available, Then a list of provided equipment is displayed.
---
## Story #6: Admin dashboard viewing
### Acceptance Criteria:
- [ ] Given an admin is logged in, When they access the dashboard, Then booking statistics are displayed.
---
## Story #7: Room maintenance scheduling
### Acceptance Criteria:
- [ ] Given notifications are enabled, When booking status changes, Then alerts are sent in real time.
---
## Story #8: Visitor booking assistance
### Acceptance Criteria:
- [ ] Given a visitor is at reception, When guided by the receptionist, Then they can access the booking system.
---
## Story #9: Booking conflict resolution
### Acceptance Criteria:
- [ ] Given no confirmation is received, When the verification expires, Then the booking is automatically cancelled.
---
## Story #10: Usage reports generation
### Acceptance Criteria:
- [ ] Given a date range is selected, When the report is generated, Then data reflects that period accurately.
